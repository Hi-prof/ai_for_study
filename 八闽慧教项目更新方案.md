# 八闽慧教项目实施规划

## 一、规划目标

本次工作不再停留在“更新方向说明”，而是明确为可执行的实施规划，确保研发、联调、测试和上线都有清晰步骤。

本次实施包含三个任务：

1. 重构并升级知识图谱生成模块。
2. 新增教师上传 PDF 文档生成知识图谱能力。
3. 修复后台管理页面新建用户时密码显示与编辑问题。

其中，知识图谱模块改造为本次实施主线，其他两项作为并行配套任务推进。

## 二、总体实施原则

为保证本次更新能够稳定落地，实施时遵循以下原则：

- 先完成架构调整，再补充功能细节，避免边开发边返工。
- 先打通主流程，再完善优化项，确保每个阶段都可验证。
- 前后端分工清晰，AI 能力统一收口到后端。
- 每个阶段都保留测试与验收节点，避免一次性堆积问题。

## 三、知识图谱模块实施规划

### 1. 实施目标

知识图谱模块本次需要完成三件事：

1. 将 AI 调用逻辑从前端迁移到后端。
2. 将一次性生成改造为智能体分阶段生成。
3. 建立“骨架图 + 轻卡片 + 深卡片”的完整生成链路。

### 2. 当前问题梳理

当前知识图谱能力存在以下问题：

- 前端直接承担 AI 调用，流程复杂后不利于统一管理、异常处理和后续扩展。
- 生成方式偏一次性输出，容易受到上下文长度限制，导致图谱结构不完整、节点内容不足。
- 虽然已经有“先图谱、后卡片”的思路，但缺少完整的阶段划分、任务编排和结果校验机制。

因此，本次实施重点不是简单调 prompt，而是完成一次生成架构升级。

### 3. 实施范围

本次知识图谱模块实施范围包括：

- 前端发起生成任务页面调整。
- 后端知识图谱生成接口与任务处理流程建设。
- 智能体分阶段生成能力接入。
- 图谱数据保存、结果回显、状态跟踪与基础校验。
- 教师确认后再落库或发布的控制流程。

### 4. 分阶段实施步骤

#### 第一阶段：现状梳理与方案定稿

目标：明确现有代码位置、调用链路和改造边界，避免开发中反复调整。

实施步骤：

1. 梳理当前知识图谱生成涉及的前端页面、接口、数据结构和 AI 调用位置。
2. 明确哪些逻辑保留在前端，哪些逻辑迁移到后端。
3. 确认知识图谱的数据模型，包括节点结构、层级关系、轻卡片字段、深卡片字段。
4. 确认生成流程状态定义，例如“待提交、生成骨架图、生成轻卡片、生成深卡片、校验中、待确认、已完成、失败”。
5. 形成统一接口设计，明确前端请求参数、后端返回结构和任务状态查询方式。

阶段产出：

- 知识图谱模块改造清单。
- 前后端接口说明。
- 图谱与卡片数据结构定义。
- 任务状态流转说明。

#### 第二阶段：AI 调用从前端迁移到后端

目标：完成架构迁移，建立后端统一处理入口。

实施步骤：

1. 前端移除直接调用 AI 的逻辑，改为仅提交生成请求。
2. 后端新增知识图谱生成任务入口，接收课程名称、教师要求、资料内容和配置参数。
3. 后端统一封装 AI 调用能力，包括模型调用、参数管理、异常捕获和日志记录。
4. 建立任务处理机制，支持按步骤执行而不是一次性返回最终结果。
5. 前端增加任务状态展示能力，能够查看当前生成进度和阶段结果。

阶段产出：

- 前端提交任务功能可用。
- 后端统一 AI 服务可用。
- 基础任务状态流转可用。

验收标准：

- 前端不再直接发起 AI 请求。
- 后端可以独立接收请求并返回任务状态。
- 生成异常时前端能看到明确失败提示。

#### 第三阶段：实现骨架图生成

目标：先让系统稳定生成课程知识图谱主干结构。

实施步骤：

1. 后端读取教师输入内容与资料摘要，整理为骨架图生成输入。
2. 智能体第一阶段只负责输出课程主结构，不展开详细内容。
3. 输出内容至少包括一级主题、二级知识点和上下级关系。
4. 后端对生成结果做基础校验，检查层级是否合法、是否存在空节点、是否存在明显重复节点。
5. 将骨架图结果存入临时数据或任务结果表，供后续卡片生成继续使用。

阶段产出：

- 可生成完整骨架图。
- 骨架图基础校验机制可用。

验收标准：

- 能基于课程信息生成一张结构清晰的思维导图。
- 图谱节点数量和层级关系明显优于当前方案。

#### 第四阶段：实现轻卡片全量生成

目标：确保图谱中的每个节点都有基础说明内容。

实施步骤：

1. 以后端已生成的骨架图节点列表为输入，逐个或分批生成轻卡片。
2. 轻卡片字段统一控制为基础信息，避免一次生成过深导致失败。
3. 轻卡片建议包含：知识点名称、简要定义、关键词、简单示例、关联知识。
4. 后端对轻卡片结果做字段完整性校验，确保每个节点都有内容返回。
5. 前端支持查看节点基础卡片信息。

阶段产出：

- 所有节点的轻卡片生成能力。
- 节点基础信息展示能力。

验收标准：

- 图谱中每个节点都能查看到基础说明。
- 卡片字段结构统一，便于后续展示和扩展。

#### 第五阶段：实现深卡片重点增强

目标：把有限资源优先投入到关键知识点，提升内容深度。

实施步骤：

1. 明确重点节点来源，支持教师手动指定和系统自动识别两种方式。
2. 智能体第三阶段只针对重点节点生成深卡片。
3. 深卡片建议包含：详细定义、核心特征、应用场景、常见问题、关联知识说明、参考内容。
4. 建立重点节点数量控制机制，避免一次选择过多导致处理时间过长。
5. 前端为重点节点展示更完整的内容区块。

阶段产出：

- 重点节点识别机制。
- 深卡片生成能力。
- 前端重点内容展示能力。

验收标准：

- 教师可以清楚区分普通节点与重点节点。
- 重点节点内容深度明显优于轻卡片。

#### 第六阶段：结果校验与教师确认流程

目标：保证输出结果可检查、可确认、可追踪。

实施步骤：

1. 后端增加生成结果校验流程，检查图谱结构完整性、卡片字段完整性和重点节点覆盖情况。
2. 对明显异常结果进行标记，例如节点过少、结构层级异常、卡片缺失。
3. 前端提供教师预览界面，支持查看骨架图、轻卡片和深卡片结果。
4. 教师确认后再执行正式保存。
5. 保存时写入完整图谱数据、卡片数据和生成来源信息，便于后续编辑与追溯。

阶段产出：

- 图谱生成结果校验机制。
- 教师确认后保存流程。

验收标准：

- 教师在保存前能够完成预览与确认。
- 系统能区分“生成完成”和“已确认保存”两个状态。

### 5. 知识图谱模块联调与测试安排

实施完成后，按以下顺序验证：

1. 测试仅输入课程名称和文字要求时的生成流程。
2. 测试带资料内容时的骨架图生成效果。
3. 测试轻卡片是否覆盖全部节点。
4. 测试重点节点深卡片是否正确生成。
5. 测试异常场景，如资料为空、AI 返回异常、部分节点生成失败。
6. 测试教师预览、确认和保存流程是否完整。

## 四、教师上传 PDF 文档生成知识图谱实施规划

### 1. 实施目标

支持教师上传 PDF 文档，并将文档内容纳入知识图谱生成流程，减少教师手动整理资料的工作量。

### 2. 分步实施

#### 第一阶段：上传能力接入

1. 前端增加 PDF 上传入口，并限制格式、大小和上传提示。
2. 后端增加 PDF 文件接收、存储和基础校验能力。
3. 对上传失败、格式错误、文件超限等情况给出明确提示。

#### 第二阶段：PDF 内容解析

1. 后端接入 PDF 文本提取能力。
2. 对提取结果做基础清洗，例如去除空白、乱码、重复片段。
3. 对超长文档做分段整理，形成可供后续生成使用的资料摘要或片段集合。

#### 第三阶段：接入知识图谱生成流程

1. 将 PDF 解析结果纳入知识图谱任务输入。
2. 在骨架图生成阶段优先参考 PDF 资料内容。
3. 在轻卡片和深卡片生成阶段引用文档中的相关信息，提升内容准确性。

#### 第四阶段：联调与验收

1. 测试普通 PDF 的上传、解析和生成链路。
2. 测试页数较多、内容较长 PDF 的处理效果。
3. 测试扫描版或文本提取效果较差的 PDF 的异常提示策略。
4. 验证 PDF 参与后图谱内容是否比纯文本输入更完整。

验收标准：

- 教师可以正常上传 PDF。
- 系统可以解析主要文本内容并用于图谱生成。
- 上传失败、解析失败时有明确反馈。

## 五、后台管理页面新建用户问题修复实施规划

### 1. 实施目标

修复后台管理页面新建用户时密码无法直接显示和编辑的问题，提升管理员操作效率。

### 2. 分步实施

1. 排查后台新建用户表单中密码字段的显示逻辑、默认值逻辑和编辑逻辑。
2. 修复密码输入框未显示、不可编辑或交互不清晰的问题。
3. 确保管理员在新建用户时可以直接输入初始密码。
4. 如页面已有密码显隐设计，补齐显示/隐藏交互；如没有，则至少保证输入框可见且可编辑。
5. 验证提交后密码字段能正确传递到后端并完成保存。

验收标准：

- 新建用户页面密码输入框正常显示。
- 管理员可以直接输入或修改初始密码。
- 提交后用户创建成功，密码字段生效。

## 六、整体实施顺序

建议按以下顺序推进，确保主线稳定、依赖清晰：

1. 完成知识图谱模块现状梳理、接口定义和数据结构定稿。
2. 完成 AI 调用从前端迁移到后端。
3. 完成骨架图生成能力。
4. 完成轻卡片全量生成能力。
5. 完成重点节点深卡片生成能力。
6. 完成结果校验、教师预览和确认保存流程。
7. 接入教师上传 PDF、解析 PDF 并纳入生成流程。
8. 修复后台新建用户密码显示与编辑问题。
9. 开展前后端联调、场景测试和问题修正。
10. 完成上线前验收与发布。

## 七、阶段交付建议

为了便于管理，可按以下方式组织交付：

- 第一批交付：后端 AI 统一入口 + 前端任务提交改造。
- 第二批交付：骨架图生成 + 轻卡片生成。
- 第三批交付：深卡片生成 + 教师确认保存。
- 第四批交付：PDF 上传解析能力。
- 第五批交付：后台用户页面问题修复 + 整体验收。

## 八、最终目标

本次实施完成后，项目应达到以下效果：

- 知识图谱生成链路从“前端直接调用 AI”升级为“后端统一调度智能体”。
- 图谱生成从一次性输出升级为分阶段生成，结果更完整、内容更可控。
- 教师可以直接上传 PDF 资料参与知识图谱生成。
- 后台管理员可以在新建用户时正常填写和编辑密码。
- 整体流程具备可执行、可验证、可上线的实施基础。
