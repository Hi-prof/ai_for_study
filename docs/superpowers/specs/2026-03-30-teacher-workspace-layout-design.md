# 教师首页工作台改版设计

## 背景

当前教师首页使用 `WorkspaceShell` 作为外壳，而进入课程后的页面使用 `teacher-course-workspace` 这一套独立布局。两者在顶部栏、左侧导航、主内容区和整体视觉节奏上差异较大，已经形成两套并行维护的页面骨架。

本次需求不是重做教师端功能，而是让教师首页在演示排版上尽量接近进入课程后的页面，同时保证后续维护成本更低。

## 目标

1. 教师首页整体布局切换为接近课程内页的视觉结构：
   - 顶部栏
   - 左侧导航
   - 右侧主内容区
2. 保留教师首页现有三个模块的业务逻辑和交互：
   - 课程管理
   - AI 教学助手
   - 账号设置
3. 提炼可复用的共享布局壳子，避免教师首页和课程详情页复制两套结构与样式。
4. 允许后续将学生端或其他工作台继续接入同一套外壳。

## 非目标

1. 不修改教师首页各模块内部的数据请求和业务逻辑。
2. 不重写课程详情页的功能模块。
3. 不在本次改版中调整导航信息架构，只做布局和视觉收敛。

## 方案选型

### 方案 A：直接复制课程详情页模板和样式到教师首页

优点：
- 上手快
- 短期视觉最像

缺点：
- 结构和样式重复
- 后续修改要同步两处
- 响应式和细节很容易逐渐漂移

### 方案 B：抽取共享工作台布局组件，教师首页和课程详情页共同复用

优点：
- 只维护一套壳子
- 教师首页和课程页天然保持一致
- 后续扩展到其他工作台更容易

缺点：
- 初次改造比复制样式多一步拆分

### 方案 C：仅在 `WorkspaceShell` 上模拟课程页视觉

优点：
- 改动范围小

缺点：
- 实际上仍保留两套布局实现
- `WorkspaceShell` 和课程详情页难以长期统一

### 结论

采用方案 B。

## 最终设计

### 1. 新增共享布局组件

新增一个共享工作台布局组件，职责只包含页面骨架，不承载业务逻辑。组件负责：

1. 顶部栏渲染：
   - 返回按钮
   - 标题/副标题区域
   - 右上角角色或用户信息区域
2. 左侧导航渲染：
   - 导航列表
   - 当前激活态
3. 右侧内容区域渲染：
   - 模块容器
   - 滚动区
   - 统一留白和边框

该组件通过 `props + slot` 驱动，支持教师首页与课程详情页传入各自的标题、导航项、头部信息和内容。

### 2. 教师首页接入共享布局

`Teacher_Mypage.vue` 不再直接使用 `WorkspaceShell`。它将改为：

1. 使用共享工作台布局组件作为外壳。
2. 继续使用当前的 `teacherWorkspaceTabs` 作为导航数据源。
3. 继续通过动态组件切换：
   - `TeacherCourses`
   - `TeacherAIAssistant`
   - `TeacherSettings`
4. 保留返回首页逻辑与当前标签持久化逻辑。

这样首页模块保持原样，只替换宿主布局。

### 3. 课程详情页迁移到共享布局

`TeacherCourseDetail.vue` 也切换到同一套共享布局组件，但保留课程详情页特有配置：

1. 课程标题来自课程信息。
2. 左侧导航使用课程详情页自己的 tab 集合。
3. 右侧内容仍由原有章节、作业、讨论、资料、学情分析等模块组成。

课程页现有针对内容模块的收口样式仍可保留，但公共外壳样式从课程详情页中移出，沉到共享布局样式文件。

### 4. 样式拆分策略

当前 `course-workspace.css` 中既包含页面骨架样式，也包含课程详情页模块覆盖样式。本次需要拆分为两层：

1. 共享布局样式：
   - 顶部栏
   - 整体背景
   - 外层容器
   - 左侧导航
   - 右侧主面板
   - 基础响应式
2. 课程详情页专属样式：
   - 各模块内容区收口
   - 按钮、卡片、图谱区域等课程页增强规则

教师首页仅依赖共享布局样式，不吃课程详情页的模块覆盖规则，避免教师首页内容被误伤。

### 5. 视觉对齐原则

教师首页与课程详情页统一以下元素：

1. 顶部栏高度、间距与操作区位置。
2. 左侧导航宽度、激活态、图标节奏。
3. 右侧内容区圆角、边框、背景与阴影。
4. 页面背景和整体留白体系。

教师首页保留以下差异：

1. 顶部标题文案仍显示“教师教学工作台”。
2. 右侧内容模块维持首页语义，不伪装成课程详情页内部模块。
3. 课程管理模块内部的统计卡片、课程列表、空状态继续沿用现有业务组件。

## 数据与组件边界

### 共享布局组件边界

共享布局组件只负责：

1. 结构布局
2. 导航渲染
3. 头部动作
4. 插槽内容承载

共享布局组件不负责：

1. 请求课程数据
2. 管理教师首页 tab 状态
3. 管理课程详情页 tab 状态
4. 任何模块内部按钮事件

### 页面组件边界

`Teacher_Mypage.vue` 和 `TeacherCourseDetail.vue` 仍分别负责：

1. 导航数据源
2. 当前 tab 状态
3. 路由跳转
4. 页面级数据加载
5. 页面级文案

## 实施步骤

1. 新建共享工作台布局组件与对应样式文件。
2. 将课程详情页中的公共外壳结构迁移到共享布局组件。
3. 将课程详情页公共骨架样式从 `course-workspace.css` 中抽离。
4. 改造 `Teacher_Mypage.vue` 接入共享布局。
5. 改造 `TeacherCourseDetail.vue` 接入共享布局。
6. 校验教师首页三个模块和课程详情页各 tab 的显示是否正常。
7. 校验桌面端和移动端响应式是否仍可用。

## 风险与处理

### 风险 1：课程详情页现有样式覆盖过深

处理：
- 只抽离公共骨架相关 class
- 保留课程详情页的模块级覆盖在专属样式文件中

### 风险 2：教师首页模块内容在新容器中尺寸异常

处理：
- 共享布局只约束外层，不对子模块内部结构做强干预
- 如需修正，只在教师首页模块样式中补充最小范围适配

### 风险 3：后续页面继续分叉

处理：
- 共享组件 API 保持清晰
- 新增工作台页面必须优先接入共享壳子，而不是复制样式

## 验证要求

1. 教师首页视觉上与进入课程后的页面具备同类结构。
2. 教师首页三个标签页切换正常。
3. 课程详情页各标签页切换正常。
4. 返回按钮、路由切换和本地状态恢复正常。
5. 页面在常见桌面宽度和窄屏宽度下没有结构性错位。
